* Emacs — Ściąga skrótów (minimalna, praktyczna)

** Pliki i bufory
- C-x C-f :: otwórz plik
- C-x b   :: przełącz bufor
- C-x C-b :: lista buforów
- C-x C-s :: zapisz plik
- C-x C-w :: zapisz jako…
- C-x k   :: zamknij bufor
- M-[     :: początek bufora (zamiennik M-<)
- M-]     :: koniec bufora (zamiennik M->)

** Dired
*** Nawigacja
n / p        → następna / poprzednia linia
RET          → otwórz plik / wejdź do katalogu
^            → katalog nadrzędny
g            → odśwież bufor (re-read directory)

*** Oznaczanie (to jest SERCE Direda)
m            → oznacz plik → wybieram zbiór → C/R/!/… → robię coś z wybranym zbiorem
u            → usuń oznaczenie
U            → usuń wszystkie oznaczenia
* t          → odwróć zaznaczenie
* s          → zaznacz wszystkie
Myśl: najpierw oznaczasz, potem robisz akcję

*** Operacje na plikach
C            → kopiuj (copy)
R            → przenieś / zmień nazwę (rename)
D            → usuń (delete)   ⚠️ od razu
d            → oznacz do usunięcia
x            → wykonaj usunięcia oznaczonych
d + x = bezpieczne kasowanie
D = „wiem co robię”

*** Tworzenie / zmiany  
+            → nowy katalog
% R          → zmiana nazw (rename by regexp) to hurtowa broń — używaj świadomie
% m          → oznacz wg regexp

*** Widok / sortowanie
s            → zmiana sortowania (name / date / size)
(            → szczegóły on/off

*** Praca z wieloma katalogami
i            → wstaw podkatalog do tego samego bufora
k            → usuń podkatalog z bufora
To robi z Direda quasi drzewo, bez magii

*** Shell / zewnętrzne polecenia
!            → uruchom polecenie shell na pliku
&            → uruchom w tle

*** Edycja nazw jak tekst (MEGA WAŻNE)
C-x C-q      → dired-toggle-read-only (tryb edycji)
C-c C-c      → zatwierdź zmiany
C-c C-k      → anuluj
To jest moment, w którym:
Dired przestaje być „menedżerem plików”,
a zaczyna być buforem do operacji na nazwach.

** Wyszukiwanie

* VC (Version Control) — skróty natywne Emacsa

** Założenie
- VC jest *wbudowane* w Emacsa
- działa niezależnie od Gita (Git to tylko backend)
- obsługa odbywa się przez prefiks: C-x v

** Prefiks
- C-x v → Version Control

** Najczęściej używane (codzienność)

- C-x v v :: commit / check-in bieżącego pliku
- C-x v = :: diff bieżącego pliku vs repo
- C-x v l :: log (historia) bieżącego pliku
- C-x v u :: revert — przywróć wersję z repo

** Nawigacja po zmianach

- C-x v n :: następna zmiana (next revision)
- C-x v p :: poprzednia zmiana (previous revision)

** Praca na repo (lekko)

- C-x v d :: vc-dir — status repo (lista plików)

** Dodawanie pliku do kontroli wersji

- C-x v i :: add / register file in VC

** Model mentalny
- VC = plik w centrum
- szybkie operacje
- niski koszt poznawczy
- Git „w tle”, nie jako temat

** Uwaga
- VC działa zawsze, jeśli Emacs widzi repo (.git)
- skróty są natywne i stabilne (część Emacsa)
